The Three-Layer Architecture

Discarding conventional cloud wrappers, Scholar Navis operates entirely locally through a rigorously engineered pipeline orchestrated by a native Qt Event Bus.

               
                   
                   

Layer 1: Secure Data Ingestion

                   

                        A dual-path pipeline guarantees data sovereignty. The persistent path independently processes local PDFs using hardware-accelerated ONNX vectorization into a local ChromaDB, enabling genuinely air-gapped environments alongside self-hosted MCP/SKILL modules for highly confidential research.                    

               
               
                   
                   

Layer 2: Logical Orchestration

                   

                        Mitigates corpus bias through automated language detection and English pre-translation. Standardized queries traverse Vector Search, Reranker scoring, in-context prompting, and the execution of academic agent tools (MCP and SKILLs) before dispatch, enforcing objective factual constraints.                    

               

Layer 3: Output Rendering

Generated outputs undergo real-time parsing via regular expressions (regex) to drive a highly interactive UI. Native support for HTTP, Mermaid diagrams, and Cite protocols enables contextual citation verification and immediate database navigation.

                   
                                           
                   

Academic Agent Tools (MCP & SKILLs)

                   

                        Deep integration with a consortium of international biological and chemical databases (NCBI, UniProt, PubChem, STRINGdb, AlphaFold, etc.) via dynamic academic agent architecture. Performs real-time fact-checking and empowers the AI to autonomously retrieve scholarly materials.                    

               

Authoritative MCP Interoperability

Direct integration with authoritative databases including NCBI, UniProt, Semantic Scholar, and Unpaywall. This architecture empowers the AI to autonomously retrieve full-text PDFs and relevant scholarly materials.

Benchmarked Extraction

Rigorously tested against structured data extraction tasks for critical plant biology genes (e.g., AMS, DYT1). Enhances manual literature curation efficiency while maintaining strict factual accuracy.

Transient Feed Processing

Stay updated with real-time RSS subscriptions from top journals. The architecture seamlessly funnels transient texts into the primary data stream for immediate analysis without permanently polluting the local ChromaDB.

Adaptive Model Routing

Designed to balance the trade-offs of base models. Mitigates the "knowledge conflict" inherent in massive pre-trained models, while accounting for the "latency penalty" associated with integrating high-accuracy native models into complex workflows.

Cross-Lingual UI Layer

A floating window accessible globally across the OS. Facilitates multi-language translation and AI polishing on highlighted text, operating seamlessly over the core execution inference layer.

Hardware Utilization

Scholar Navis maximizes local inference speeds dynamically routing tasks via CUDA (NVIDIA), DirectML (AMD/Intel), or CoreML (Apple Silicon).

Minimum Configuration

  • Memory (RAM) 8 GB
  • Graphics (GPU) Not Required
    (CPU-only Inference)
  • Disk Space ~2 GB (Lite Models)
  • Models Arctic-XS / BGE-Base

Recommended (Pro)

  • Memory (RAM) 16 GB - 32 GB
  • Graphics (VRAM) NVIDIA 6GB+
    (RTX 3060/4060+)
  • Disk Space ~10 GB (High Precision)
  • Models BGE-M3 / Qwen2-7B-Embed

Note: High-performance Reranker models (e.g., Gemma-9B architecture) may incur substantial latency penalties or require 20GB+ VRAM for optimal execution speed.
